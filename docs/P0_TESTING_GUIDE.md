### 3️⃣ 测试 API 端点 (可选)

使用浏览器访问后端文档测试：

1. 打开 **http://localhost:8001/docs**
2. 找到 P0 相关端点：
   - `POST /api/v1/music/continue` - 歌曲续写
   - `POST /api/v1/music/extend-structure` - 结构扩展
   - `POST /api/v1/subtitles/recognize` - 字幕识别
   - `GET /api/v1/subtitles/health` - 健康检查
3. 点击 "Try it out" 测试

**快速测试命令** (在终端):
```bash
# 测试歌曲续写
curl -X POST http://localhost:8001/api/v1/music/continue \
  -H "Content-Type: application/json" \
  -d '{"song_id":"demo-001","continue_from":60,"duration":60}'

# 测试结构扩展
curl -X POST http://localhost:8001/api/v1/music/extend-structure \
  -H "Content-Type: application/json" \
  -d '{"song_id":"demo-001","structure":"Intro-Verse-Chorus"}'

# 测试字幕健康检查
curl http://localhost:8001/api/v1/subtitles/health
```

## ⚠️ 已知问题

### CSS 导入警告
前端启动时有 CSS 警告：
```
[vite:css] @import must precede all other statements
```
**影响**: 不影响功能，仅样式可能有偏差  
**解决**: 下次优化 `src/index.css`

### Mock 模式
以下功能处于 Mock 模式：
- 歌曲续写 → 返回示例响应
- 自动字幕 → 生成模拟字幕
- MV 模板应用 → console.log 输出
- 转场效果应用 → console.log 输出

**真实化方案**:
- 接入 Mureka API 实现真实续写
- 安装 Whisper 实现真实字幕识别
- 集成 FFmpeg.wasm 实现模板/转场应用
